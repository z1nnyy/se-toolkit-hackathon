## Slide 2. Context

### Text for the slide

- End users: Innopolis University students, staff, and campus visitors
- Problem: during busy hours people often need to go to the counter just to see the current menu
- Product idea: a Telegram bot and admin panel for browsing and updating the Cava menu in real time

### Short speech

The end users are students, staff, and visitors on campus. The main problem is that during busy hours the menu is not easy to access, because people have to go to the counter and check it physically. My solution is a Telegram bot connected to an admin panel, so users can see the menu instantly and staff can keep it updated.

## Slide 3. Implementation

### Text for the slide

- Built with `FastAPI`, `SQLite`, `React`, and `aiogram`
- Version 1:
  - backend with database
  - Telegram bot for menu browsing
  - admin panel for editing dishes and prices
- Version 2:
  - login/password protection
  - super-admin access management
  - Russian/English menu
  - generated menu images in Telegram
  - seasonal menu collections
  - Docker deployment
- Addressed feedback:
  - easier bot navigation
  - safer admin access
  - more realistic menu structure

### Short speech

I built the product with `FastAPI` for the backend, `SQLite` for the database, `React` for the admin panel, and `aiogram` for the Telegram bot. In Version 1 I implemented the core functionality: storing dishes, showing them in Telegram, and editing them in the admin panel. In Version 2 I improved the product by adding authentication, staff management, bilingual menu support, generated menu images, seasonal menu collections, and Dockerized deployment. I also improved the bot interaction so it is easier to use and does not spam the chat.
